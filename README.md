# Веб-приложение "Интернет - магазин"

## Описание:      
    веб-приложение, которое содержит три отдельных приложения приложения: 
    "catalog" - список товаров интернет магазина;
    "blog_app" - интернет ресурс с полезными статьями;
    "users" - регистрация и авторизация пользователей, а также настройки доступа
    для авторизированных пользователей.
    Данное приложение разработано с помощью фреймворка Django.
    Подключена PostgreSQL для работы в проекте.

## Описание программы:

1. #### Приложение catalog 
##### Описание:
    Содержит шесть веб-страниц "Главная страница", "Информация о товаре", "Добавление нового товара", 
    "Продукты из категории", "Подтверждение удаления товара" и "Контакты".
    Организовано переключение по соответствующим ссылкам между этими страницами.
    Для данных веб-страниц созданы контроллеры, для которых настроена маршрутизация.
    Для стилизации страниц используется Bootstrap. 
    Подключена PostgreSQL для работы в проекте.
    Созданы модели Product и Category. Для этих моделей создана миграция.
    Описание моделей:
    Product:
    * наименование,
    * описание,
    * изображение,
    * категория,
    * стоимость,
    * дата создания,
    * дата последнего изменения,
    * поставщик,
    * публикация на сайте.

    Category:
    * наименование,
    * описание.

    Создан суперпользователь.
    Модели Product и Category зарегистрированы в админке.
    Созданы фикстуры для моделей Category и Product.
    Создана кастомная команда add_products для добавления тестовых продуктов.
    Команда предварительно удаляет все существующие данные из базы перед добавлением
    новых продуктов.
    
    Создайн новый контроллер и шаблон для отображения страницы с подробной информацией о товаре. На этой странице 
    показана вся информация о товаре.
    В шаблон главной страницы добавлен код для отображения списка товаров с помощью цикла. 
    Чтобы карточки товаров выглядели одинаково, отображаемое описание обрезано до первых 37 символов.
    Создан базовый шаблон, который будет включает общие элементы страниц. 
    Также созданы подшаблоны для главного меню, для нижнего колонтитула и для заголовка в верхней части страницы.    
    Для работы с каталогом интернет магазина реализован полный CRUD с использованием CBV и модуля django.forms.
    Реализуйте валидацию формы, которая проверяет отсутствие запрещенных слов в названии и описании продуктов.
    Добавлена кастомная валидация для поля 'price', которая запрещает устанавливать стоимость товара меньше 0.    
    Добавлена стилизация форм для продуктов, используя метод __init__.
    Добавлены кастомные права к модели продукта:
    can_unpublish_product — может отменять публикацию продукта.
    Добавлено поле владельца (owner) к модели продукта. Поле  связано с моделью пользователя через ForeignKey.
    Создана сервисная функция, которая возвращает список всех продуктов в указанной категории. 
    Реализуйте отдельное представление и шаблон для отображения этой информации на сайте.
    Настроено кеширование для страниц "Главная страница", "Информация о товаре" и "Продукты из категории".

2. #### Приложение blog_app 
##### Описание:
    Содержит четыри веб-страницы "Главная страница", "Информация о блоге", "Создание блога" и 
    "Подтверждение удаления блога".
    Организовано переключение по соответствующим ссылкам между этими страницами.
    Для данных веб-страниц созданы контроллеры, для которых настроена маршрутизация.
    Для стилизации страниц используется Bootstrap.        
    Подключена PostgreSQL для работы в проекте.
    Создана модель Blog. Для этой модели создана миграция.
    Описание модели:    
    * заголовок,
    * содержимое,
    * превью (изображение),
    * признак публикации (булевое поле),    
    * дата создания,
    * количество просмотров.
    
    Модель Blog зарегистрирована в админке.    
    
    Создайн контроллер и шаблон для отображения страницы с подробной информацией о блоге. На этой странице 
    показана вся информация о блоге.
    В шаблон главной страницы добавлен код для отображения списка блогов с помощью цикла. На этой странице 
    отображаются только те статьи, которые имеют положительный признак публикации.
    Для работы с блогом реализован полный CRUD с использованием CBV и модуля django.forms. 
    При открытии отдельной статьи увеличивать счетчик просмотров.
    После успешного редактирования записи пользователь перенаправляется на просмотр этой статьи.
    Реализуйте валидацию формы, которая проверяет отсутствие запрещенных слов в названии и содержании блога.
    Добавлена стилизация форм для блога, используя метод __init__.

3. #### Приложение users 
##### Описание:
    Содержит две веб-страницы "Регистрация пользователя" и "Авторизация пользователя".
    В сервисе реализованы функции авторизации и регистрации пользователя. После регистрации пользователя на почту
    приходит приветственное письмо с ссылкой, пройдя по которой пользователь получает токен и завершает регистрацию.
    Для авторизации необходимо ввести адрес электронной почты и пароля.
    Создана модель пользователя User наследующаяся от AbstractUser. Для этой модели создана миграция.
    Описание модели User:   
    * электронная почта,
    * аватар (изображение),
    * телефон,
    * страна.
    
    Электронная почта используется как поле для авторизации.
    Создана команда csu для создания суперпользователя.    
    
    Доступ к страницам, связанным с управлением продуктами и блогом, открыт только для авторизованных пользователей.
    Общедоступной остаётся только страница просмотра блога и списка товаров.
    Создана группа «Модератор продуктов» и ей назначены следующие права:
    * Разрешение на отображение продуктов на сайте (can_unpublish_product).
    * Разрешение на удаление любого продукта.
    Настроена проверка наличия прав у пользователя при попытке отменить публикацию или удалить продукт.
    При создании продукт автоматически привязывается к авторизованному пользователю.
    Кнопка "Подробнее" отображается у каждого авторизованного пользователя.
    Кнопки "Подробнее" и "Удалить" отображаются у модератора.    
    Кнопки "Подробнее", "Редактировать" и "Удалить" отображаются только у поставщика продукта.
